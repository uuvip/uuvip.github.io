解决openwrt市面上常见的upnp失效不可用
第一修改 /etc/config/upnpd 文件
我的固件只需在 /etc/config/upnpd里加上option external_iface 'WAN'这句代码重启就可以了
解决方案：使用winscp或ssh vi命令修改/etc/config/upnpd文件
config upnpd 'config'
option external_iface 'WAN'
option internal_iface 'lan'
option port '5000'
option upnp_lease_file '/var/run/miniupnpd.leases'
option enabled '1'
option uuid '4bd6f6b5-61d0-4ddf-a170-26c318494fa9'
option presentation_url 'http://192.168.8.1'
option download '1024'
option upload '512'
解决方案：使用winscp或ssh vi命令修改文件
第二改miniupnpd的配置文件
/tmp/etc/miniupnpd.conf
ext_ifname=pppoe-wan
ext_ifname6=pppoe-wan
listening_ip=br-lan
enable_natpmp=yes
enable_upnp=yes
secure_mode=yes
system_uptime=yes
force_igd_desc_v1=no
ext_perform_stun=no
ipv6_disable=no
bitrate_down=8388608
bitrate_up=4194304
lease_file=/var/run/miniupnpd.leases
presentation_url=http://192.168.8.1
port=5000
uuid=4bd6f6b5-61d0-4ddf-a170-26c318494fa9
allow 1024-65535 0.0.0.0/0 1024-65535 #Allow high ports
deny 0-65535 0.0.0.0/0 0-65535 #Default deny
最后重启路由
